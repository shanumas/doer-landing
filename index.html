How it works

Three steps.
Zero babysitting.

01
🔗

Connect

Connect your GitHub account and select a repository. doer watches for open issues labelled doer and respects dependency order automatically.

→
02
✅

Approve

An Adaptive Card lands in your Microsoft Teams channel. One tap to approve or deny — you stay in control of what gets worked on.

→
03
🚀

Done

doer spins up an isolated sandbox, clones your repo, writes the fix, commits, and opens a pull request. Every decision is logged for your team to audit.

Features

Built for real teams.

🤖

Model agnostic

Works with OpenAI, Azure OpenAI, Anthropic, Ollama, or any OpenAI-compatible endpoint. Your data goes directly to your provider — no middleman.

📦

Sandboxed execution

Every task runs in an isolated Docker container that is destroyed after the PR is opened. Your host system is never touched.

🧠

Dependency aware

doer reads issue bodies for "depends on #N" and "blocked by #N" patterns and always picks the unblocked issue first.

📋

GitHub Projects v2

Reads both labelled issues and GitHub Projects boards. Priority order from your project board is respected.

🔍

Full audit trail

Every shell command, AI decision, and file change is streamed live to your dashboard and stored for governance and compliance.

🏢

Enterprise ready

OAuth via GitHub and Microsoft Entra ID. Self-hosted with Docker Compose. Runs inside your own infrastructure — no data leaves your network.

Open Source

Free forever.
Own your stack.

doer is MIT licensed. No usage fees, no API paywalls, no vendor lock-in. Fork it, extend it, deploy it in your own cloud. Contributions welcome.
